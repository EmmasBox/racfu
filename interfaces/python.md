# Python

The following Python interface is provided to facilitate exploitation of RACFu by Python callers.
{: .fs-6 .fw-300 }

&nbsp;

The Python distribution of RACFu may be installed from [PyPi](https://pypi.org/project/racfu/) using `pip`.

&nbsp;

```shell
pip install racfu
```

The Python distribution of RACFu may also optionally be downloaded and installed from [GitHub](https://github.com/ambitus/racfu/releases).

## `racfu.racfu()`

```python
def racfu(request: dict) -> SecurityResult:
```

#### 📄 Description

Make a **Security Request** to RACF.

#### 📥 Parameters

* `request` <br>
  A dictionary that defines the **Security Request** to perform following the [RACFu Request JSON Format](../../request_format).

#### 📤 Returns
* `SecurityResult`<br>
  An [Object](#racfusecurityresult) that describes the **Security Request** that was made. 

#### ❌ Raises
This method does **NOT** raise any exceptions.

#### 💻 Example

The following example issues a **Profile Extract** request to extract the profile data for the **z/OS userid** `SQUIDWRD`.

###### Python Script
```python
from racfu import racfu

request = {
    "admin_type": "user",
    "operation": "extract",
    "profile_name": "SQUIDWRD"
}

result = racfu(request)
```

## `racfu.SecurityResult`

```python
class SecurityResult:
    def __init__(
            self,
            request: dict,
            raw_request: bytes | None,
            raw_result: bytes | None,
            result: dict | None
    ):
```

#### 📄 Description

&nbsp;

{: .note }
> _The `result` **Class Attribute** is what most users and exploiters will typically care about. The `request`, `raw_request`, and `raw_result` **Class Attributes** are more useful for **Debugging**._

&nbsp;

An **Object** that describes a **Security Request** that was made.

#### 📒 Class Attributes

* `request`<br>
  A dictionary that defines the **Security Request** that was made following the [RACFu Request JSON Format](../../request_format).

* `raw_request`<br>
  A bytes object containing the exact contents of the **Raw Request** made to the corresponding **RACF Callable Service**.

* `raw_result`<br>
  A bytes object contains the exact contents of the **Raw Result** returned by the corresponding **RACF Callable Service** that was called.

* `result` <br>
  A dictionary that contains the **Post-Processed Security Result** following the [RACFu Result JSON Format](../../result_format).

#### 💻 Example

The following example makes a **Profile Extract** request to extract the profile data for the **z/OS userid** `SQUIDWRD`, and then displays all of the **Class Attributes** returned in the corresponding `SecurityResult` object.

###### Python Script
```python
from racfu import racfu
import json

request = {
    "admin_type": "user",
    "operation": "extract",
    "profile_name": "SQUIDWRD"
}

# The racfu() function always returns a SecurityResult object.
result = racfu(request)

# These class attributes are more useful for debugging.
print("RACFu Request JSON:")
print(json.dumps(result.request, indent=2))
print("Raw Request to R_Admin:")
print(result.raw_request)
print("Raw R_Admin Result:")
print(result.raw_result)

# This is the class attribute most will care about.
print("RACFu Result JSON:")
print(json.dumps(result.result, indent=2))
```
